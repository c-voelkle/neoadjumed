Project Canvas

Modernisierung Adjumed Analyze
Erstellt von: Curtis Völkle (CV)
Wo/Wann: Adjumed Services AG · März 2026
Varianten: A (Baseline) & B (AI-beschleunigt)
🎯 Zweck

Die bestehende Adjumed-Analyze-Plattform basiert auf AngularJS (End-of-Life) + jQuery 3.7.1 mit einer VB.NET WebForms-Middleware (ca. 250.000–400.000 LOC) – Technologien ohne aktive Weiterentwicklung. Die gesamte Geschäftslogik (Filter, Abfragen, Exporte) ist im Code-Behind eingebettet, ohne API-Schicht. Das Projekt modernisiert die Analyseplattform zu einer zeitgemässen Architektur (React 18 + TypeScript → .NET 8 REST API → SQL Server) mittels Strangler-Fig-Pattern. Damit wird die Grundlage geschaffen für langfristige Wartbarkeit, zukünftige Erweiterungen (Mobile, KI-Funktionen) und eine verbesserte Benutzererfahrung für Kliniken schweizweit.

🌊 Umfeld

Rückenwind

  • React bereits in Collect/Validate im Einsatz
  • .NET 8 als strategische Richtung etabliert
  • Analyze-DB (flache Tabellen) deutlich einfacher als Collect-DB (EAV)
  • Publish-Prozess bleibt unverändert – kein Risiko
  • Frontier-KI-Coding-Tools erstmals produktionsreif (Claude Code Agent Teams)

Gegenwind

  • IW-Verfügbarkeit: nur 2–3 h/Wo. für Reviews + IIS
  • Legacy-Codebasis: 250–400k LOC VB.NET ohne Dokumentation
  • Schweizer Datenschutz: sorgfältige Trennung Mandanten
  • Parallelbetrieb altes/neues System nötig
🏁 Etappenziele
  • Gate 0a: VB.NET Codebase analysiert; Geschäftslogik dokumentiert (Pseudocode, Flowchart); ADR genehmigt
  • Gate 0b: 3 PoC-Endpunkte (z.B. /auth/token, /query, /filters) korrekt + performant; Mandantentrennung sicher; Repository-Interface für Datenbank-Entkopplung; Go/No-Go Demo
  • Gate 1: Produktionsreife API; React-Frontend Filtern → Analysieren → Export End-to-End
  • Gate 2: Dashboard, Charts, Pilotergebnisse 3–5 Kliniken; Feedbackbericht
  • Gate 3: 100% Migration inkl. User Assets (Filtersets, Ausgabesets, etc.); Altsystem abgeschaltet

Richtungsweisend

  • Gate 0b entscheidet über Weiterführung des gesamten Projekts
✨ Qualität

Quantitativ

  • API-Antwortzeit (P95): Preview ≤ 300 ms, Standard ≤ 1,5 s
  • Gefühlte Ladezeit ≤ 50 % des Altsystems
  • Exportgenerierung ≤ 10 Sek.
  • Verfügbarkeit ≥ 99,5 %
  • Support-Aufwand ≤ 50 % der aktuellen Analyze-Anfragen

Qualitativ

  • Klartext-Filter ohne Feldcode-Kenntnisse
  • Alle Reports (HSM, Uro, Endo, SBCDB, AQC) identische Ergebnisse
